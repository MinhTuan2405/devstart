## Bảo mật AI Apps Script bắt đầu từ đâu?

Bảo mật AI Apps Script bắt đầu từ nguyên tắc tối thiểu: không hardcode API key, không log dữ liệu nhạy cảm, chỉ gửi dữ liệu cần thiết cho model, giới hạn OAuth scopes và luôn review trước khi deploy trigger, web app hoặc workflow tự động.

AI automation có rủi ro cao hơn script thường vì dữ liệu có thể rời khỏi Google Workspace để đi tới AI provider. Một dòng prompt tưởng như vô hại có thể chứa email khách hàng, nội dung hợp đồng, token truy cập hoặc thông tin nội bộ. Vì vậy bảo mật không phải phần làm sau, mà phải nằm trong thiết kế ngay từ đầu.

## Mental model: secret, dữ liệu và quyền là ba lớp khác nhau

Secret là API key, token, webhook secret hoặc credential. Dữ liệu là nội dung người dùng, sheet, email, tài liệu, file Drive. Quyền là OAuth scopes và quyền truy cập của người chạy script.

Bạn cần bảo vệ cả ba lớp:

- Secret không được nằm trong code, log hoặc prompt gửi cho AI.
- Dữ liệu gửi cho model phải được tối thiểu hóa và ẩn danh khi có thể.
- Quyền OAuth phải đủ dùng, không rộng hơn nhu cầu.

## Không hardcode API key

API key nên nằm trong script properties hoặc hệ thống secret bên ngoài. Với Apps Script project nhỏ, `PropertiesService` là lựa chọn thực tế.

```javascript
function getAiApiKey() {
  const apiKey = PropertiesService.getScriptProperties().getProperty('AI_API_KEY');
  if (!apiKey) {
    throw new Error('Missing AI_API_KEY');
  }
  return apiKey;
}

function getSafeConfigForLog() {
  const props = PropertiesService.getScriptProperties();
  return {
    provider: props.getProperty('AI_PROVIDER') || 'gemini',
    model: props.getProperty('AI_MODEL') || 'unknown',
    dryRun: props.getProperty('DRY_RUN') === 'true',
  };
}
```

Lưu ý function log chỉ trả thông tin an toàn, không trả API key. Nếu bạn dùng clasp và git, hãy kiểm tra `.gitignore` và tránh commit credential local.

## Redact dữ liệu trước khi log

Không phải dữ liệu nào cũng cần log nguyên văn. Email, số điện thoại, tên khách hàng, mã đơn hàng hoặc nội dung dài nên được che hoặc cắt ngắn.

```javascript
function maskEmail(email) {
  const parts = String(email || '').split('@');
  if (parts.length !== 2) return 'invalid-email';
  return parts[0].slice(0, 2) + '***@' + parts[1];
}

function previewText(value, maxLength) {
  return String(value || '').slice(0, maxLength || 120);
}

function logSafeFeedback(item) {
  console.log({
    rowNumber: item.rowNumber,
    email: maskEmail(item.email),
    messagePreview: previewText(item.message, 80),
  });
}
```

Với production, hãy hạn chế log nội dung prompt đầy đủ. Nếu cần debug, dùng dữ liệu giả hoặc log bản đã ẩn danh.

## Sanitize dữ liệu trước khi gửi cho AI

Một thói quen tốt là tạo function biến object nội bộ thành payload an toàn cho AI. Function này chỉ giữ field cần thiết.

```javascript
function sanitizeLeadForAi(lead) {
  return {
    need: String(lead.need || '').slice(0, 1000),
    category: String(lead.category || ''),
    language: String(lead.language || 'vi'),
  };
}

function buildLeadPrompt(lead) {
  const safeLead = sanitizeLeadForAi(lead);
  return [
    'Summarize this lead request in Vietnamese.',
    'Do not infer personal identity.',
    '',
    'Need: ' + safeLead.need,
    'Category: ' + safeLead.category,
  ].join('\n');
}
```

Trong ví dụ này, email, phone và internal note không được đưa vào prompt. Đây là cách kiểm soát dữ liệu bằng code thay vì chỉ dựa vào lời nhắc trong prompt.

## Dữ liệu nào không nên gửi cho AI?

- API key, access token, refresh token, webhook secret.
- Email, số điện thoại, địa chỉ hoặc mã khách hàng nếu không cần cho tác vụ.
- Dữ liệu tài chính, y tế, pháp lý hoặc hợp đồng nếu chưa có chính sách xử lý rõ.
- Nội dung Gmail riêng tư không liên quan đến tác vụ.
- Toàn bộ spreadsheet khi chỉ cần một vài cột.
- File Drive có quyền hạn chế mà người dùng không kỳ vọng sẽ gửi ra ngoài.

Nếu không chắc dữ liệu có được gửi không, hãy hỏi người sở hữu dữ liệu hoặc dùng bản ẩn danh.

## Kiểm soát OAuth scopes

Apps Script có thể tự suy luận scopes, nhưng bạn nên xem lại manifest khi dự án nghiêm túc. Scope càng rộng, rủi ro càng cao nếu script bị lạm dụng.

```json
{
  "timeZone": "Asia/Ho_Chi_Minh",
  "runtimeVersion": "V8",
  "exceptionLogging": "STACKDRIVER",
  "oauthScopes": [
    "https://www.googleapis.com/auth/spreadsheets.currentonly",
    "https://www.googleapis.com/auth/script.external_request"
  ]
}
```

Ví dụ trên phù hợp hơn cho script chỉ làm việc với spreadsheet hiện tại và gọi API ngoài. Đừng thêm scope Gmail hoặc Drive nếu workflow không dùng.

## Lỗi thường gặp về bảo mật

- Dán API key vào prompt nhờ AI debug.
- Deploy web app public nhận prompt tự do nhưng không kiểm soát abuse.
- Gửi toàn bộ spreadsheet cho model để tóm tắt vài dòng.
- Không phân quyền sheet chứa dữ liệu nhạy cảm.
- Dùng scope rộng hơn nhu cầu, ví dụ toàn Drive khi chỉ cần sheet hiện tại.
- Log response AI đầy đủ có chứa dữ liệu người dùng.
- Không có dry run hoặc giới hạn request khi test.

## Bài tập thực hành

Viết function `sanitizeLeadForAi(lead)` chỉ giữ `message`, `category` và `language`, bỏ `email`, `phone`, `company`, `internalNote`. Tạo test function chứng minh output không còn email hoặc phone. Sau đó viết `buildLeadPrompt` dùng object đã sanitize, không dùng object gốc.

## Câu hỏi thường gặp về bảo mật AI Apps Script

### Có nên dùng script properties cho secret không?

Có thể dùng cho dự án nhỏ và nội bộ. Với hệ thống doanh nghiệp hoặc nhiều người dùng, cân nhắc Secret Manager, backend proxy hoặc cơ chế quản lý secret tập trung.

### Có nên gửi dữ liệu thật cho AI khi debug không?

Không nên. Hãy dùng dữ liệu giả, đã ẩn danh hoặc đã cắt ngắn. Nếu lỗi chỉ cần format dữ liệu, dữ liệu giả thường đủ.

### OAuth scope có liên quan AI không?

Có. Script tích hợp AI thường đọc Gmail, Drive, Sheets hoặc gọi API ngoài. Scope quyết định script được phép truy cập gì, nên phải giới hạn ở mức cần thiết.

### Prompt có được xem là dữ liệu nhạy cảm không?

Có thể. Prompt thường chứa dữ liệu người dùng hoặc thông tin nghiệp vụ. Đừng log prompt đầy đủ mặc định, và đừng gửi prompt production cho AI debug nếu chưa ẩn dữ liệu.

## Tóm tắt và bước tiếp theo

Bảo mật là nền tảng của mọi AI automation. Hãy tách secret khỏi code, sanitize dữ liệu trước khi gửi cho AI, log an toàn và kiểm soát OAuth scopes. Bài tiếp theo sẽ bắt đầu gọi Gemini API bằng `UrlFetchApp` trong Apps Script.
