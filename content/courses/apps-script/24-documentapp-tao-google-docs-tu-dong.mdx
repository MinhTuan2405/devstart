`DocumentApp` là service dùng để tạo và chỉnh sửa Google Docs bằng Apps Script. Bạn dùng nó khi muốn biến dữ liệu từ Sheets thành báo cáo, biên bản, phiếu xác nhận, hợp đồng nháp hoặc tài liệu cá nhân hóa mà không cần soạn thủ công từng file.

## DocumentApp trong Apps Script là gì?

Nếu `DriveApp` quản lý file và folder, `DocumentApp` quản lý nội dung bên trong Google Docs. Nó cho phép bạn tạo document mới, mở document có sẵn, thêm heading, paragraph, table, list item và chỉnh một số định dạng cơ bản. Trong automation thực tế, `DocumentApp` thường đi cùng `SpreadsheetApp`, `DriveApp` và `MailApp`: đọc dữ liệu từ Sheets, tạo Docs, lưu trong Drive, gửi link qua email.

Mô hình tư duy là: một document có `Body`; trong body có các phần tử như paragraph, table, list item. Bạn không viết HTML vào Docs. Bạn gọi method như `appendParagraph`, `appendTable`, `setHeading`. Sau khi chỉnh xong, nên gọi `saveAndClose()` trước khi gửi link hoặc lấy file để xuất PDF.

## Tạo Google Docs đầu tiên

Đoạn mã sau tạo tài liệu mới, thêm tiêu đề, thời gian tạo và một đoạn mô tả. Code dùng chuỗi ASCII trong nội dung mẫu để tránh lỗi copy.

```javascript
function createSimpleDocument() {
  const doc = DocumentApp.create('Automated report');
  const body = doc.getBody();

  body.appendParagraph('Automated report')
    .setHeading(DocumentApp.ParagraphHeading.HEADING1);

  body.appendParagraph('Created at: ' + new Date().toISOString());
  body.appendParagraph('This document was generated by Apps Script.');

  doc.saveAndClose();
  Logger.log('Doc URL: ' + doc.getUrl());
}
```

Sau khi chạy, document nằm trong Drive của tài khoản chạy script. Nếu bạn cần lưu vào folder cụ thể, hãy tạo document trước rồi dùng `DriveApp.getFileById(doc.getId())` để di chuyển hoặc thêm vào folder.

## Thêm bảng dữ liệu vào Docs

Báo cáo thường cần bảng. `appendTable` nhận mảng hai chiều, trong đó mỗi mảng con là một dòng. Dữ liệu nên là chuỗi hoặc giá trị có thể hiển thị rõ ràng.

```javascript
function createDocumentWithTable() {
  const doc = DocumentApp.create('Sales report');
  const body = doc.getBody();
  const rows = [
    ['Metric', 'Value'],
    ['Orders', '42'],
    ['Revenue', '12500000'],
    ['Refunds', '2'],
  ];

  body.appendParagraph('Sales report')
    .setHeading(DocumentApp.ParagraphHeading.HEADING1);
  body.appendParagraph('Summary generated from spreadsheet data.');
  body.appendTable(rows);

  doc.saveAndClose();
  Logger.log(doc.getUrl());
}
```

Nếu lấy dữ liệu từ Sheets, hãy chuẩn hóa trước khi đưa vào bảng. Ví dụ ngày tháng nên format thành chuỗi, số tiền nên format rõ ràng, ô trống nên thay bằng `N/A` hoặc chuỗi rỗng có chủ đích.

## Tạo báo cáo Docs từ dữ liệu Sheets

Ví dụ thực tế: sheet `Metrics` có cột `Metric` và `Value`. Script đọc dữ liệu, tạo document, thêm bảng, rồi ghi URL vào sheet `ReportLog`.

```javascript
function createReportFromSheet() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const metricSheet = ss.getSheetByName('Metrics');
  const logSheet = ss.getSheetByName('ReportLog');
  const values = metricSheet.getDataRange().getDisplayValues();

  const doc = DocumentApp.create('Report ' + Utilities.formatDate(new Date(), Session.getScriptTimeZone(), 'yyyy-MM-dd'));
  const body = doc.getBody();

  body.appendParagraph('Daily metrics report')
    .setHeading(DocumentApp.ParagraphHeading.HEADING1);
  body.appendParagraph('Generated at: ' + new Date().toISOString());
  body.appendTable(values);

  doc.saveAndClose();
  logSheet.appendRow([new Date(), doc.getId(), doc.getUrl()]);
}
```

`getDisplayValues()` thường tốt hơn `getValues()` khi đưa vào document vì nó lấy dạng hiển thị trong sheet, ví dụ tiền tệ hoặc ngày tháng đã được format.

## Mở và cập nhật Google Docs có sẵn

Không phải lúc nào bạn cũng tạo document mới. Nếu có template hoặc tài liệu cố định, bạn có thể mở bằng ID.

```javascript
function appendNoteToExistingDoc() {
  const docId = 'PASTE_DOC_ID_HERE';
  const doc = DocumentApp.openById(docId);
  const body = doc.getBody();

  body.appendParagraph('Update at: ' + new Date().toISOString());
  body.appendParagraph('New note added by Apps Script.');

  doc.saveAndClose();
}
```

Với template phức tạp, bạn có thể copy file bằng `DriveApp`, mở bản copy bằng `DocumentApp`, rồi thay placeholder. Đây là hướng nâng cao nhưng rất hữu ích cho hợp đồng hoặc phiếu xác nhận.

## Khi nào dùng DocumentApp?

- Tạo báo cáo định kỳ từ dữ liệu Sheets.
- Tạo biên bản họp, phiếu xác nhận hoặc hợp đồng nháp.
- Cá nhân hóa tài liệu cho nhiều người nhận.
- Tạo bản nháp nội dung để team review.
- Chuẩn hóa tài liệu thay vì để mỗi người tạo thủ công khác nhau.

Nếu bạn cần điều khiển Google Docs ở mức rất sâu như batch update phức tạp, Docs API nâng cao có thể phù hợp hơn. Nhưng với người mới, `DocumentApp` đủ cho rất nhiều use case.

## Lỗi thường gặp với DocumentApp

- Quên `saveAndClose()` trước khi gửi link hoặc xuất PDF.
- Tạo quá nhiều document khi test mà không dọn Drive.
- Không chuẩn hóa dữ liệu Sheets trước khi đưa vào bảng.
- Nhầm nội dung Docs với HTML và cố chèn HTML string.
- Không kiểm soát quyền chia sẻ tài liệu sau khi tạo.
- Không lưu document ID, khiến lần sau khó cập nhật đúng tài liệu.

## Bài tập thực hành

Tạo sheet `Metrics` có cột `Metric`, `Value`. Tạo sheet `ReportLog` có cột `CreatedAt`, `DocId`, `DocUrl`. Viết function đọc dữ liệu từ `Metrics`, tạo Google Docs mới, thêm tiêu đề, thời gian tạo và bảng dữ liệu. Sau đó gọi `saveAndClose()` và ghi ID, URL vào `ReportLog`.

## Câu hỏi thường gặp về DocumentApp

### DocumentApp có sửa được Docs có sẵn không?

Có. Dùng `DocumentApp.openById(documentId)` nếu tài khoản chạy script có quyền truy cập tài liệu đó.

### Có thể chèn ảnh không?

Có, nhưng bạn cần Blob ảnh từ Drive hoặc nguồn khác. Người mới nên thành thạo paragraph và table trước vì đây là nền tảng của báo cáo tự động.

### Có thể xuất PDF không?

Có thể kết hợp `DriveApp` để lấy file Docs và chuyển Blob sang PDF. Đây là bước nâng cao thường dùng khi gửi báo cáo qua email.

### Có nên tạo document mới mỗi lần chạy không?

Tùy mục tiêu. Báo cáo theo ngày nên tạo file mới và lưu log. Tài liệu trạng thái cố định có thể mở file cũ và cập nhật.

## Tóm tắt và chuyển tiếp

`DocumentApp` giúp biến dữ liệu thành Google Docs tự động. Hãy hiểu document body, dùng method như `appendParagraph` và `appendTable`, lưu ID để truy vết và gọi `saveAndClose()`. Bài tiếp theo sẽ học `CalendarApp`, service giúp tạo sự kiện lịch tự động từ dữ liệu có sẵn.
